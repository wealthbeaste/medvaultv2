API Reference

Base URL: https://your-railway-url.up.railway.app

All requests require an Authorization: Bearer <token> header unless marked as public.

Authentication

POST/api/auth/registerCreate organisation + owner account
POST/api/auth/loginGet JWT token
GET/api/auth/meCurrent user profile

Login Request

{
  "email": "user@pharmacy.com",
  "password": "your_password"
}

Login Response

{
  "token": "eyJhbGciOi...",
  "user": {
    "id": 1, "name": "Owner", "email": "user@pharmacy.com",
    "role": "owner", "orgId": 1, "pharmacyId": 1
  }
}

Inventory

GET/api/inventory?page=1&limit=50&search=paraList drugs with pagination
POST/api/inventoryAdd new drug
PUT/api/inventory/:idUpdate drug
DELETE/api/inventory/:idDelete drug
GET/api/inventory/alertsLow stock + expiring drugs
POST/api/inventory/adjustStock adjustment with audit
GET/api/inventory/:id/purchase-historyGRN + PO history for a drug

Sales

GET/api/sales?page=1&limit=50List sales with pagination
POST/api/salesRecord new sale (transactional)
GET/api/activityStaff activity log

Response Format

All endpoints return JSON. Successful responses include the data directly. Errors follow this format:

{
  "success": false,
  "error": "Human-readable error message",
  "code": "VALIDATION_ERROR",
  "field": "email"
}
